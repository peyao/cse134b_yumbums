Conformance

  • a
    • IE: When clicking on an 'a' tag with the download attribute present, it opens up a new browser window that views the file specified in the href.
    • Firefox: When clicking on an 'a' tag with the download attribute present, it opens up a new browser window that views the file specified in the href.
    • Chrome: When clicking on an 'a' tag with the download attribute present, it downloads the file specified by the href to the user's computer.
  • abbr
    • IE: When a title attribute is present, the abbreviation is still displayed the same as without the title.
    • Firefox: When a title attribute is specified, it underlines the text of the abbreviation with a dotted line.
    • Chrome: When a title attribute is present, the abbreviation is still displayed the same as without the title.
  • rtc
    • IE: Text inside the 'rtc' element is positioned to the side of the text inside the 'rb' element next to it, but the text is still displayed with a small font unlike Chrome.
    • Firefox: Text inside the 'rtc' element is positioned above the text of the 'rb' element which corresponds to the examples in the HTML specification.
    • Chrome: Text inside the 'rtc' element is positioned to the side of the text inside the 'rb' element next to it.
  • bdi
    • IE: The words 'apple store' are displayed from right to left instead of left to right. It seems like IE doesn't support the 'bdi' tag. The validator also gave a warning saying that the 'bdi' is not supported by browsers.
    • Firefox: The sentence was displayed correctly implying support for the 'bdi' tag.
    • Chrome: The sentence was displayed correctly implying support for the 'bdi' tag.
  • embed
    • IE11: For Quicktime videos, displays a player but does not play content.
    • Edge: For Quicktime videos, nothing displayed at all, not even a 'Plugin missing' box.
    • Firefox: For Quicktime vidoes, 'Plugin missing' box.
    • Chrome: For Quicktime videos, 'Plugin missing' box.
  • video
    • IE11: 'Invalid source' and 'Error: Unsupported video type or invalid file path'.
    • Edge: 'Invalid source' and 'Error: Unsupported video type or invalid file path'.
    • Safari: Controls show up but video does not play.
    • Firefox: OK
    • Chrome: OK
    • Conclusion: IE, Edge, and Safari doesn't support WebM.
  • meter
    • IE11: The display bar for the "meter element" is missing.
    • Edge: The display bar for the "meter element" is missing.
  • ouput
    • IE11: The output for the "output element" is missing and slider is non-functional.
    • Edge: The output for the "output element" is missing and slider is non-functional.
